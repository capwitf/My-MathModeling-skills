# 章节格式控制

## 用途

用于保持竞赛论文各章节的视觉形态一致。它控制章节形状、分问标签、标点和间距，不替代模型或证据判断。

## 全局分问标签规则

在 `问题重述`、`问题分析与思路概述` 和简短分问总结中，使用加粗标签、中文全角冒号和一个空格。

Markdown 形式：

```markdown
**问题一：** [正文从这里开始。]
**问题二：** [正文从这里开始。]
**问题三：** [正文从这里开始。]
```

LaTeX 形式：

```tex
\textbf{问题一：} [正文从这里开始。]
\textbf{问题二：} [正文从这里开始。]
\textbf{问题三：} [正文从这里开始。]
```

规则：

- 使用中文全角冒号 `：`，不要使用 ASCII 冒号 `:`。
- 加粗标签后保留一个可见空格。
- 不要写成 `问题一:` 或 `问题一 ：`。
- 除非该章节很长或官方模板要求，否则不要把每个分问段都改成小节。
- 标签要平行：若使用 `问题一`，后文继续使用 `问题二`、`问题三`，不要混用 `第一问`。

## 问题重述

默认形状：`问题背景 -> 问题提出`。当赛题有明确现实场景时，这更接近官方优秀论文的写法。

```markdown
### 1.1 问题背景

[背景段：1-2段，交代现实场景、核心矛盾、与建模有关的对象和指标。]

[可选背景图：仅使用已准备好的真实图片；插入前替换为真实文件路径，没有图片则删除本行。]

图1 [图注：说明图中对象与赛题背景的关系；若未插图则删除本图注。]

### 1.2 问题提出

**问题一：** [用自己的话说明输入、限制、输出。]
**问题二：** [说明在问题一基础上的新增目标、约束或交付物。]
**问题三：** [说明进一步优化、评价、预测或推广任务。]
```

质量控制：

- 论文模板允许二级标题时，使用 `1.1 问题背景` 和 `1.2 问题提出`。
- 背景段不超过 2 个短段。
- 背景图是可选项；只有当图能更清楚展示真实对象、场景、几何关系、流程或决策背景时才放。
- 背景图注应具体，例如 `图1 现实中的烟幕弹运用`，不要只作装饰。
- 每个分问段不超过 3 句。
- 不要逐句复制原题。
- 不要为了填充版面使用素材感强、无关的背景图。

## 问题分析

默认形状：总分。总述段说明全题建模分解；每个分问段按照 `目标 -> 数学归类 -> 难点 -> 路线 -> 输出/验证` 写。

```markdown
[总述段：1段，说明全题可分解为哪些建模任务，以及各问之间的递进关系。]

**问题一：** 问题一要求[交付物]，本质是[预测/评价/优化/仿真/参数辨识]问题。该问题的主要难点在于[难点]。考虑到[题目特征]，本文采用[模型或算法]，输出[结果表/图/方案/参数]，并通过[验证方式]检验。

**问题二：** 在问题一得到[结果/参数/约束]的基础上，问题二进一步考虑[新增因素]。该问题可归结为[数学问题类型]，难点在于[难点]。本文采用[模型或算法]处理[关键关系]，输出[结果]，并通过[验证方式]检验。
```

质量控制：

- 问题分析通常控制在 0.5-1 页。
- 每个分问用一个自然段，不写成项目符号列表。
- 每问必须回答“本质是什么数学问题”，不能只复述题目要求或算法名称。
- 不要以"经过分析可知"开头。
- 不要只写"采用某模型"，必须说明数学类型和选择理由。
- 若使用问题关系图，将其放在分问段之后，并补一句结论。

## 分问题模型章节

正文主体优先使用一级章节：

```text
五、问题一模型的建立与求解
六、问题二模型的建立与求解
七、问题三模型的建立与求解
```

章节内部只为真实写作任务设置编号小节：

```text
5.1 本问建模思路
5.2 模型建立
5.3 求解方法
5.4 结果与分析
5.5 本问结论
```

除非官方模板强制要求，不要设置一个庞大的通用 `模型建立与求解` 章节，再把所有子问题藏在下面。
