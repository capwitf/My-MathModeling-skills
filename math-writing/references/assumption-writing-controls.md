# 模型假设与评价控制

## 用途

用于防止 `模型假设` 和 `模型评价` 写成长、空、泛的装饰性文字。它控制长度、焦点和措辞。

## 模型假设

默认形状：4-6 条编号假设。只有当题目包含多个物理子系统时，才放宽到最多 8 条。

```text
1. 将[对象A]、[对象B]及[关键目标]视为[质点/刚体/理想几何体]进行分析。
2. 不考虑[空气阻力/风速/外部扰动/测量误差]对[运动/判断/结果]的影响。
3. 假设[状态/速度/参数]在[时间范围/有效期]内保持不变。
4. 假设[响应/调整/通信]时间相对于[任务时间尺度]可忽略。
5. 假设[形状/范围/覆盖区域]可近似为[几何体]，其[半径/密度/能力]在[条件]内不变。
```

质量控制：

- 一条假设通常只写一句。
- 说清理想化对象和忽略因素。
- 正文假设要紧凑；详细合理性放在模型准备、灵敏度分析或附录中。
- 强假设应在检验或灵敏度讨论中被回收。
- 每条假设必须服务一个公式、约束、分布、边界或验证动作；不能只为"简化计算"占位。
- 不要写"假设数据真实可靠"。
- 不要只写"为简化计算"，除非下一句说明该简化为什么可接受。

## 模型评价

默认形状：

```text
11.1 模型的评价

模型的优点：
1. [难点转译]：[模型如何把题目中最难判断的现实条件转化为状态变量、约束组或判定函数，为什么比简单经验规则更可靠]，见[式/表/图/前问结果/支撑材料]。
2. [判定闭环]：[模型如何形成"状态定义 -> 判定函数 -> 结果输出 -> 回代检验"的链条，使结论可计算、可核查]，见[式/表/图/前问结果/支撑材料]。
3. [跨问递推/约束显式化]：[前一问的结果如何成为后一问的输入、约束或基准；关键物理/业务限制如何进入目标函数或约束条件]，见[式/表/图/前问结果/支撑材料]。

模型的缺点（缺点写适用边界）：
1. [现实因素缺失]：[说明忽略了什么，以及可能影响哪个结论。]
2. [表达或实现代价]：[说明符号、篇幅、计算量或参数依赖的代价。]

11.2 模型的推广

本文模型在[理想/给定]条件下建立。若推广到[现实/更大规模/新场景]，需要补充[环境扰动/新约束/参数标定/数据来源]，并重新检验[可行性/灵敏度/鲁棒性]。
```

证据钩子：每条优点至少对应一个正文可见证据来源，可写为"见式[编号]"、"见表[编号]"、"见图[编号]"、"由问题X结果可知"或"见支撑材料[文件名]"。运行记录编号只保留在支撑材料或登记表中，不直接写入正文优点句。没有证据钩子的句子不得写成模型优点。

## 框架优点的含义

框架优点关注模型结构，而不是数值结果好不好。这里的 `框架优点` 对应英文检索词 framework-level advantages。

框架优点不是把"坐标、变量、时间"列齐，而是说明模型结构解决了题目里的关键判断。不要只写统一坐标；必须说明哪个现实难点因此变得可计算、可复核、可迁移。

优先从这些角度写：

- 难点转译：把[遮蔽/可行性/覆盖/风险/匹配/调度冲突]从文字判断转化为状态变量、约束组或判定函数，避免只用"距离小于阈值"、"得分较高"这类粗糙规则。
- 判定闭环：形成"状态定义 -> 判定函数 -> 结果输出 -> 回代检验"的链条，评委能看出每个结论由哪个条件推出。
- 跨问递推：问题一得到的[轨迹/参数/分类/排名/可行域]直接作为问题二或问题三的输入、约束或基准，避免多个模型彼此孤立。
- 约束显式化：把[物理限制/业务规则/安全边界/资源容量]写进目标函数、约束或判定条件，而不是只在文字中说明。
- 简化有边界：核心理想化假设在灵敏度、局限或推广中被回收，说明结论在什么条件下仍成立。
- 证据可追溯：正文结果能对应到式[编号]、表[编号]、图[编号]、前问结果或支撑材料；运行记录编号留在支撑材料或登记表中。图像必须说明机制、几何关系、临界区间或方案差异，不只是提高可读性。

避免：

- "模型结果较好"；
- 没有规模或证据的"算法效率较高"；
- "具有较强优越性"；
- 没有同构场景的"适用范围广"；
- 只写"统一了坐标、变量和时间框架"，却不说明哪个难关系变得可计算；
- 只写"图像直观清晰"，却不说明图像证明了什么机制、区间、路径或对比；
- 只写"模块化结构便于复用"，却不说明哪一个输出进入了后续哪一问；
- 把每个小公式都列成优点。

好模式：

```text
1. 本文没有将[核心难点]简化为[粗糙规则]，而是将其表示为[状态变量/约束组/判定函数]，使[遮蔽/可行性/覆盖/风险]能够在每个[时间步/场景/对象]下计算和回代检验。
2. 问题一得到的[轨迹/参数/判定结果/可行域]直接作为问题二的[输入/约束/基准]，使多问之间形成"建模--求解--再优化"的递推关系，而不是彼此独立的模型堆叠。
3. 模型把[物理/业务限制]写入[目标函数/约束/判定条件]，因此评委可以直接追踪结论由哪些条件推出；图[编号]进一步显示[机制/临界区间/方案差异]。
```

证据钩子：上述三条应分别能追溯到式[编号]、表[编号]、图[编号]、前问结果或支撑材料；运行记录编号只用于支撑材料或登记表。

坏模式：

```text
1. 本文模型具有较好的效果。
2. 本文算法先进，结果比较理想。
```
