# 数学建模评分维度

## 用途

用这 6 个维度审查每一篇论文、软件或实物模型。复查时不要只说“写得不好”或“模型一般”，而要把问题映射到具体评分维度。

## 六项评分维度

1. 数学模型的实际原型或来源阐述的充分性
2. 数学模型归结的准确性与合理性
3. 数学模型过程的正确性与理论水平
4. 数学模型的创新性
5. 模型或相关软件的应用前景
6. 数学模型的整体表述

## 复查口径

## 可读性 Overlay

可读性不是第 7 个官方维度，默认映射到维度 6“数学模型的整体表述”；当阅读成本掩盖现实原型、数学归结或求解验证时，也会牵连维度 1-3。审查时不要只问“内容是否正确”，还要问“疲劳评委是否能在一遍阅读中看清路线、证据和边界”。

高风险信号：

- 公式堆砌：长公式连续出现，但没有任务含义、变量解释、验证位置或结论句。
- 逻辑跳跃：从题意到模型、从模型到算法、从结果到结论之间缺少桥接句。
- 图表混乱：图表顺序、编号、单位、图后结论或正文引用让读者难以核验。
- 浮动体割裂：图片压在页底或表格跨页处理不当，使引入文字、图表和结论分散到不同阅读位置。
- 页面越界：图表、长公式或表格超出正文宽度，导致边缘被裁切、进入页边距或只能靠过度缩放勉强放下。
- 阅读成本过高：核心结果藏在大段推导、密集表格或附录细节中，评委无法快速定位。

可读性问题可以是 P3 润色项；如果它遮住核心模型、结果表、验证或必答交付物，应按 P2 处理。

### 1. 实际原型或来源阐述

检查论文是否讲清楚现实问题从哪里来、业务背景是什么、数据代表什么、模型服务什么决策。

常见失分：

- 背景泛泛而谈
- 数据字段没有解释
- 现实对象和数学变量脱节
- 没说明模型对应的实际系统边界

### 2. 模型归结的准确性与合理性

检查题目是否被正确转化为数学问题。重点看目标函数、决策变量、约束边界是否对应题意。

常见失分：

- 小问交付物漏掉
- 目标函数和题目目标不一致
- 把重要业务约束当成次要项
- 假设过强，破坏现实意义

### 3. 模型过程的正确性与理论水平

检查推导、算法、求解、验证是否正确。重点看量纲、边界、守恒、可行性、收敛性和结果复核。

常见失分：

- 方程左右量纲不一致
- 功率平衡、质量守恒、流量守恒不清
- 约束不闭合
- 结果无法复现
- 验证环节单薄

可见得分证据：

- 边界条件检查、回代检查或极端情形检查。
- 参数扰动、步长敏感性或多种子稳定性。
- 基准方案、对照方案或有限规模穷举验证。
- 支撑材料能复现论文的核心数值和图表。

### 4. 模型创新性

检查是否有恰当的新指标、新约束、新组合模型、新验证角度或任务解释框架。创新必须服务题目，不是堆复杂名词。

常见失分：

- 只套模板模型
- 创新点没有数学表达
- 创新点没有结果证明
- 方法复杂但不服务题目

可见得分证据：

- `innovation_ledger.csv` 中有 baseline、改动组件、证据需求、消融/对照和验证状态。
- 创新 claim 能追溯到结果表、图表证据、run record 或模型检验。
- 至少说明一个被拒绝的伪创新或弱创新，避免为复杂而复杂。

### 5. 应用前景

检查模型是否可落地、可解释、可迁移，是否给出实际管理建议或软件化输出。

常见失分：

- 只有理论结果，没有执行方案
- 没有说明适用范围
- 没有讨论局限性
- 软件或代码输出不能支持论文结论

### 6. 整体表述

检查摘要、结构、图表、公式、符号、参考文献、排版和语言是否让评委低成本验收。

常见失分：

- 摘要没有核心数值
- 图表小或无单位
- 公式编号混乱
- 符号前后不一致
- 结论没有证据链
- 排版降低可信度

可见得分证据：

- 首页或摘要有核心数值，且能追溯到正文证据。
- 图表标题、表头、坐标轴和符号表单位清楚。
- 每个关键图表附近有结论说明。
- 提交格式、页数、附件和匿名性按官方规则或用户提供规则检查。
- `claim_ledger.csv`、`result_registry.csv`、`figure_evidence.csv` 与正文一致。
- `submission_checklist.md` 记录规则、页数、附件、AI 使用披露和匿名硬闸；`final_submission_manifest.md` 记录最终 PDF、支撑材料、匿名检查和规则来源。

## 评分映射快捷表

| 证据 | 优先映射维度 |
| --- | --- |
| `model_quality_review.md` 证明主路线、基准模型、变量约束、结果表和验证闭环完整 | 1, 2, 3, 6 |
| 机理流程、守恒关系、边界条件 | 1, 2, 3 |
| 变量、目标、约束与题意逐项对应 | 2 |
| 基准比较、穷举验证、对照实验 | 3, 4 |
| 鲁棒性、敏感性、极端场景解释 | 3, 5 |
| 支撑材料可复现核心结论 | 3, 6 |
| 摘要数值可追溯、图表单位清楚 | 6 |
| 创新 ledger + baseline/ablation evidence | 4 |
| figure evidence + 图后结论句 | 3, 6 |
| final submission manifest + 匿名检查 | 6 |

## High-risk optimization mappings

Use these mappings when prioritizing review findings:

| Risk | Primary scoring dimensions |
| --- | --- |
| Feasibility wording is wrong, such as packaging a relaxed or violated solution as strictly feasible | 2, 3 |
| Required results are not carried by complete tables, or core numbers appear only in figures/prose | 6 |
| Solver, run, convergence, or reproducibility records are missing | 3, 5 |
| Claimed innovation has no strong result, baseline, ablation, or sensitivity support | 4 |
| Extension indicators do not serve the problem target or are not included in result tables | 2, 6 |
