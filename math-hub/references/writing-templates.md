# 国奖式论文章节段落模板

## 用途

给 AI 提供高教社杯/CUMCM 风格论文的段落级写作结构。不是内容指导，而是"这一段该怎么组织、开头怎么写、分几段、每段干什么"的模板。

国奖式论文不靠章节名堆满，而靠每个子问题形成"分析--模型--求解--结果--检验--结论"闭环。官方格式规范只硬性约束摘要页、正文、附录、匿名、引用和支撑材料；正文小节顺序应服从赛题表达和评委扫描效率。

## 推荐章节顺序

### 摘要专用页

1. 论文题目
2. 摘要
3. 关键词

摘要应放在最终写作阶段完成，但模板位置在论文第一页。摘要必须直接给方法、结果和最强量化结论，不能写成引言。每个分问摘要句覆盖交付物、方法角色、关键结果、验证或边界；数值预算按最终答案、阈值/方案、提升率、误差/鲁棒性排序。

### 正文主体

1. 问题重述
2. 问题分析与思路概述
3. 模型假设与符号说明
4. 模型准备（复杂题可用）
5. 问题一模型的建立与求解
6. 问题二模型的建立与求解
7. 问题三模型的建立与求解
8. 模型检验、灵敏度分析与结果讨论
9. 模型评价、改进与推广
10. 参考文献
11. 附录

可接受变体：若变量和假设很多，可采用"问题重述--模型假设--符号说明--问题和数据的初步分析--问题一模型的建立与求解--问题二模型的建立与求解"。若多个子问题共用坐标系、基础方程、判定函数、离散化或算法框架，增加"模型准备"集中说明。无论采用哪种变体，都不要放目录；正文前置部分应短，主要篇幅留给分问建模、结果和验证。

## 一、问题重述

### 结构

总—分结构。第一段概括全题背景和总目标，后续每段对应一个子问题。

### 模板

```
第一段（总）：
[一句话交代现实背景和核心矛盾] + [一句话说明本题要求解决什么类型的问题] + [一句话点明需要交付的核心成果类型（方案/模型/策略/评价体系）]。

第二段起（分）：
**问题一：** [用自己的语言重新表述该子问题的输入、限制和输出，不照抄原文] + [点明输入数据和约束条件] + [说明需要交付什么形式的结果（表/图/方案/数值）]。

**问题二：** ...（同上格式）
```

### 要求

- 不要照搬赛题原文，要用建模语言重新表述。
- 每个子问题的重述要体现"输入→处理→输出"的逻辑。
- 总段不超过 4 句，分段每段不超过 3 句。
- 禁止出现"本文将建立模型进行求解"这类空话。

---

## 二、模型假设

### 结构

编号列表，每条假设一个条目。主文假设应短，不写成论证段。详细合理性、影响范围和检验位置放到模型准备、灵敏度分析或附录。

### 模板

```
1. 将[对象A]、[对象B]及[关键目标]视为[质点/刚体/理想几何体]进行分析。
2. 不考虑[空气阻力/风速/外部扰动/测量误差]对[运动/判断/结果]的影响。
3. 假设[状态/速度/参数]在[时间范围/有效期]内保持不变。
4. 假设[响应/调整/通信]时间相对于[任务时间尺度]可忽略。
5. 假设[形状/范围/覆盖区域]可近似为[几何体]，其[半径/密度/能力]在[条件]内不变。
```

### 要求

- 假设数量默认 4-6 条，复杂物理系统最多 8 条。
- 每条假设通常只写 1 句。
- 强假设（可能影响结论的）要在检验或灵敏度分析中回收，不一定在假设节展开长依据。
- 假设按影响范围从大到小排列。
- 每条假设必须服务一个公式、约束、分布、边界或验证动作。
- 禁止写"假设数据真实可靠"这类无信息量的假设。

---

## 三、问题分析与思路概述

### 目标

问题分析是评委判断"是否真正针对题目建模"的关键入口。它回答"这个子问题在数学上是什么问题、为什么选这条路线、会输出什么"，而不是直接跳到公式。

### 结构

国奖式写法通常有两种：

- 独立短节：每个子问题一段，总长度控制在 0.5-1 页。
- 嵌入式写法：在每个"问题X模型的建立与求解"开头放 3-5 句分析。

独立短节优先采用总分结构：先用一段总述说明全题建模分解和各问递进关系，再每个子问题一段。分问标签统一写成 `**问题一：** `、`**问题二：** `，即问题编号加粗，中文全角冒号，冒号后一个空格。内部按"目标--数学归类--难点--路线选择--预期输出与验证"组织。

### 模板

```
【总述段】（1段，2-4句）
"本题可分解为[任务1]、[任务2]和[任务3]三个递进建模任务。"
+ "其中，问题一的[结果/参数/约束]将作为问题二的[输入/基准/约束]，问题三进一步[优化/评价/推广]。"

【单问分析段】（1段，4-6句）
"**问题一：** 问题一要求[交付什么决策/结果/方案]。"
+ "从数学结构看，该问题可归结为[预测/评价/优化/分类/网络流/仿真/参数辨识/多目标决策]问题。"
+ "该问题的核心难点在于[变量多/非线性/不确定性/耦合/组合爆炸/数据稀疏/约束可行性/...]。"
+ "考虑到[问题特征1]和[问题特征2]，本文选择[模型类型]作为主要建模工具。"
+ "选择理由：[一句话说明模型与问题特征的匹配关系]。"
+ "预期输出为[结果表/最优方案/预测值/排名/...]，验证方式为[基准对比/回代/敏感性/...]。"
```

### 句式库

路线选择理由：
- "由于目标函数[特征]且决策变量为[类型]，问题本质为[数学问题类型]。"
- "数据呈现[时序/空间/网络/层次]结构，适合采用[对应方法]进行建模。"
- "题目要求的交付物为[完整调度表/排名/路径/...]，需要[精确求解/启发式搜索/枚举+筛选]。"
- "问题涉及[不确定性来源]，因此在确定性模型基础上引入[鲁棒/随机/场景]处理。"

### 要求

- 每问的分析段必须出现在该问模型建立之前。
- 每问必须回答"本质是什么数学问题"；模型选择从机制、变量和约束推出，不能由算法名推出。
- 必须说明"为什么不用更简单的方法"或"为什么不用更复杂的方法"。
- 禁止写"经过分析，我们决定使用XX模型"而不给理由。
- 如果多问共用一个模型框架，在第一问详细说明，后续问只说差异。
- 如果已经有独立"问题分析与思路概述"章节，分问模型小节开头只写承接句，不重复整段分析。
- 复杂多问题可加入问题关系图，说明各问之间的输入、约束、基准和递进关系。

## 四、模型准备（复杂题可用）

### 目标

把多个子问题共用的坐标系、数据预处理、基础方程、核心判定函数、离散化和算法框架提前说明，避免在每个子问题里重复铺垫。

### 模板

```
【坐标系与数据预处理】（1段）
为统一描述[对象]的[位置/状态/时间变化]，建立[坐标系/时间步/索引集合]。原始数据经过[单位统一/坐标转换/缺失处理/离散化]后，形成[矩阵/表/参数集]。

【共用变量与基础关系】（1-2段）
定义[共用变量]表示[含义]，其单位为[单位]。根据[几何/物理/业务逻辑]，得到基础关系式[公式编号]，该式将在问题[编号]中用于[用途]。

【核心判定函数或评价指标】（1段）
为判断[遮挡/可行性/匹配/风险/约束满足]，构造[函数名]。当[条件]满足时，[函数]取[值/状态]；否则取[值/状态]。

【离散化与算法框架】（1段）
由于[连续空间/组合规模/非线性]导致直接求解困难，本文采用[网格化/时间离散/分层搜索/启发式算法]。粒度或参数设置为[值]，后续通过[敏感性/误差/收敛]检验其合理性。
```

### 要求

- 只有共用内容才放在模型准备；只服务单问的变量和公式放回对应子问题。
- 模型准备中的每个公式都要说明会在哪些问题中使用。
- 建议配坐标系、流程或问题关系图，但图后必须有结论句。
- 若采用离散化，检验节必须说明步长、粒度或采样数量对结论的影响。

---

## 五、问题一模型的建立与求解（每个子问题作为一级章节）

### 结构

每个子问题通常独立为一级章节，例如"问题一模型的建立与求解"、"问题二模型的建立与求解"。章节内部按"本问建模思路→模型构建→求解方法→结果呈现→结果分析→本问小结"展开。国奖论文的主体不是算法介绍，而是每一问都让评委快速看到变量、目标函数、约束、算法、结果表和回答。

### 模板

```
【一级标题】
"问题一模型的建立与求解"

【本问建模思路】（1段，2-4句）
[本问的核心决策/预测/评价目标是什么] + [输入数据的特征和挑战] + [为什么选择这类模型（一句话定性理由）]。若已有"模型准备"，本段只写本问新增变量、约束或目标。

【模型构建段】（1-3段）
第一段：[定义决策变量/状态变量，说明物理含义和取值范围]。
第二段：[给出目标函数，解释每一项的物理意义]。
第三段：[列出约束条件，每个约束说明对应什么现实限制]。

公式前铺垫句式：
- "为刻画[现象]，引入[变量]表示[含义]，建立如下[关系/目标/约束]："
- "考虑到[物理/业务约束]，[变量]需满足："
- "将[现实需求]转化为数学表达："

公式后解释句式：
- "其中，[变量]表示[含义]，单位为[单位]；[参数]由[来源]确定。"
- "式(X)的物理意义为：[一句话解释这个公式在说什么]。"

【求解方法段】（1段，3-5句）
[选用什么算法/求解器] + [为什么选它（问题特征匹配）] + [参数、步长、样本数、停止条件或求解器设置] + [计算规模说明]。

【结果呈现段】
[插入结果表或图] + [表/图标题要包含对象、场景、指标]。主结果表写清场景、单位、方案编号、关键指标和结论句。

【结果分析与本问小结】（1-2段）
第一段：[从表/图中提取关键数值] + [比较不同方案/场景/参数的差异] + [给出量化结论]。
第二段（如有）：[解释结果背后的机理] + [讨论边界情况或异常值] + [联系回问题目标说明实际意义]。本问末句形成"结果 + 证据 + 决策意义"闭环。

结果分析句式：
- "由表X可见，[方案A]在[指标]上达到[数值][单位]，较[基准/方案B]提升[百分比/差值]，表明[机理解释]。"
- "当[参数]超过[阈值]时，[指标]出现[变化趋势]，这是因为[物理/逻辑原因]。"
- "综合[指标1]和[指标2]，[最终结论/推荐方案]。"
```

## 八、模型检验、灵敏度分析与结果讨论

### 结构

先说检验什么、怎么检验，再给结果，最后下结论。

### 模板

```
【检验设计段】（1段）
"为验证模型[X]的[稳定性/可靠性/鲁棒性]，对[参数/权重/随机性/阈值]进行[扰动/交叉验证/多种子测试]，观察[指标]的变化情况。"

【检验结果段】
[插入灵敏度图/对比表] + [图表标题写清参数范围和观察指标]。

【检验结论段】（2-3句）
"当[参数]在[范围]内变化时，[指标]的变化幅度为[数值]，[排名/符号/可行性]保持不变。"
+ "[结论：模型对该参数不敏感/敏感，需要注意...]。"
+ "[如有]当[参数]超出[范围]时，[什么会发生]，因此模型适用范围为[...]。"
```

## 九、模型评价、改进与推广

### 结构

优点--不足--推广，三块式。优点优先评价"框架上的优点"，不是泛泛夸结果好。所谓框架优点，是指模型怎样完成难点转译、判定闭环、跨问递推和约束显式化：把现实难点变成可计算结构，把状态判断变成判定函数，把前一问结果传给后一问，并让关键限制进入目标函数或约束条件。不要只写统一坐标；图像必须说明机制、临界区间、几何关系或方案差异。

### 模板

```
【优点段】
模型的优点：
1. [难点转译]：[说明模型如何把题目的核心难点转化为状态变量、约束组或判定函数，为什么比简单经验规则更可靠]，见[式/表/图/前问结果/支撑材料]。
2. [判定闭环]：[说明"状态定义--判定函数--结果输出--回代检验"如何让结论可计算、可核查]，见[式/表/图/前问结果/支撑材料]。
3. [跨问递推/约束显式化]：[说明前一问结果如何进入后一问输入、约束或基准；关键现实限制如何写入目标函数或约束]，见[式/表/图/前问结果/支撑材料]。

【缺点段】
模型的缺点：
1. [现实因素缺失]：[说明忽略了什么，以及可能影响哪个结论。]
2. [表达或实现代价]：[说明符号、篇幅、计算量或参数依赖的代价。]

【推广段】
本文模型在[理想/给定]条件下建立。若推广到[现实/更大规模/新场景]，需要补充[环境扰动/新约束/参数标定/数据来源]，并重新检验[可行性/灵敏度/鲁棒性]。
```

证据钩子：每条优点至少对应正文可见证据，如式[编号]、表[编号]、图[编号]、前问结果或支撑材料；运行记录编号只保留在支撑材料或登记表中。没有证据钩子的句子不得写成模型优点。优点写结构价值，缺点写适用边界。

### 要求

- 优点必须指向模型框架、约束表达、判定函数、可解释性或复用性。
- 写框架优点时，不要只写"统一坐标/变量/时间"，必须说明哪个现实难点因此被转译为可计算结构。
- 写图像优点时，图像必须说明机制、临界区间、几何关系或方案差异，不能只写"直观清晰"。
- 写优点时必须挂接证据钩子：式[编号]、表[编号]、图[编号]、前问结果或支撑材料至少出现一种；运行记录编号只用于支撑材料或登记表。
- 不要把"结果较好"写成模型优点；结果属于结果分析，评价节应解释为什么框架合理。
- 缺点必须具体，不能写"模型较为简单"这种空话。
- 禁止写"时间有限，未能充分验证"。

## 十、模型推广与应用建议（默认并入模型评价）

### 目标

本节用于补足模型评价中的推广边界。竞赛论文默认写 1 段，2-4 句；只有题目明确要求应用建议时，才单列小节。评委看的是推广是否同构、需要改什么参数/数据/约束，以及边界是否诚实，不看长篇展望。

### 结构

短结构：同构场景 → 适配项 → 失效边界。

### 模板

```
本文模型可迁移至[同构场景]，因为该场景同样具有[相同数学结构/约束类型/决策目标]。推广时需要重新标定[参数]，替换[数据来源]，并根据[新场景限制]调整[约束/目标函数/算法粒度]。当[关键假设或边界条件]不成立时，模型需要引入[修正机制]后才能使用。
```

### 句式库

- "该模型可用于[同构场景]，前提是[关键假设]仍成立。"
- "迁移时主要调整[参数/数据/约束]，模型结构不必重写。"
- "若[新场景]引入[新限制]，需在现有模型中增加[约束类型]。"
- "当[条件]超出[范围]时，当前假设失效，应改用[修正机制/替代模型]。"

### 要求

- 推广场景必须和本题的数学结构有明确对应关系，不能天马行空。
- 不要求跨领域类比；没有自然同构场景时，可以只写推广边界。
- 参数适配必须具体到改什么参数/数据/约束。
- 不虚构量化收益；只有题目或结果提供依据时才写收益估算。
- 边界段必须诚实，指出模型在什么条件下会失效。
- 禁止写"可推广到更多领域"这类空话。
- 禁止写"具有广阔的应用前景"而不给具体场景。
- 推广建议应能让读者判断：如果要用这个模型，需要准备什么、改什么、在哪些条件下不能直接用。

## 摘要专用页模板

### 结构

背景定位（1句）→ 各问方法和结果（每问1-2句）→ 验证/鲁棒性（1句）→ 最强结论收束（1句）。

### 模板

```
"针对[问题核心]，本文建立了[模型总称]。"

```

### 要求

- 参见 `math-abstract` SKILL 的完整规则。
- 每个数字都要能在正文中找到来源。
- 模型名要具体到本题，不能写"优化模型"。

---

## 通用禁止句式

以下句式在任何章节都不应出现：

- "本文将建立数学模型进行求解"（空话）
- "经过分析可知"（没说分析了什么）
- "结果较为理想"（没有量化标准）
- "我们认为/我们觉得"（主观）
- "众所周知"（不需要写的东西）
- "为了简化计算，假设..."（假设需要合理性依据，不是为了简化）
- "由于时间有限"（评委不关心你的时间）
- "进一步优化后效果更好"（没说优化了什么、好了多少）

---

## 通用过渡句式库

### 子问题之间

- "在问题一确定了[X]的基础上，问题二进一步考虑[Y]的影响。"
- "问题一的[结果/参数]作为问题二的[输入/约束]，据此建立..."

### 模型建立到求解

- "上述模型为[问题类型]，具有[特征]，适合采用[算法类型]求解。"
- "由于目标函数[特征]且约束[特征]，选用[具体算法]进行求解。"

### 结果到验证

- "为检验上述结果的可靠性，从[角度]进行验证。"
- "上述最优解是否对[参数]敏感，需要进一步检验。"

### 单问内部承接

- "基于上述变量定义，建立目标函数如下："
- "在满足式(X)-(Y)约束的前提下，求解结果如表Z所示。"
- "表Z的结果表明[结论]，其物理机理为[解释]。"
