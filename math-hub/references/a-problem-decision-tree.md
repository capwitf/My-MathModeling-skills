# A 题题型快速分诊表

## 用途

本表用于高教社杯 A 题或等价综合题的开局分诊。它不替代精读题面，也不直接决定最终模型；它只帮助总控在前 20-40 分钟判断主线、辅线、第一批数据任务和验证口径。

## 使用原则

1. 先看题面要求的最终交付物，再看数据形态，最后选模型工具。
2. 若题面同时命中多类信号，选一个主线，另一个作为辅线。例如“预测需求后安排车辆”应是“预测后决策”主线，路径调度作为后续优化模块。
3. 每条路线都必须先做可解释基准模型，再做改进模型。
4. 分诊结论必须写成“候选路线”，不能把候选工具写成最终模型。
5. 分诊输出必须包含至少一张预期结果表和一个验证动作。
6. 每条候选路线给出 `match_confidence`。若题面机制不能清楚命中表中任一类，进入 `low-confidence triage`，使用 `new_mechanism_route`，不强行归类。

## 低置信分诊规则

`low-confidence triage` 不是失败，而是新题型保护机制。出现下列情况时，不强行归类：

- 题面核心对象、现实机制或最终交付物与表中路线都只弱相关；
- 能命中多个路线，但没有一个能推出完整结果表和验证动作；
- 近年案例库只能提供局部结构，不能解释当前题的主要机制；
- 用户或题面要求的输出不是常见优化、评价、仿真或预测结果。

此时先做 30-60 分钟机制审计：锁定交付物、变量和可观测数据，写出最小 baseline，确定第一个 validation action，再决定是否形成新的主线。

## 开局输出模板

```text
match_confidence:
候选主线:
辅线模块:
题面命中信号:
若低置信: new_mechanism_route / 机制审计任务
必须交付结果表:
第一版基准模型:
第一批代码任务:
验证闭环:
暂不采用的路线:
原因:
```

## 组合路线提示

- “预测 + 调度”：先锁定预测误差如何进入优化，不要把两部分写成互不相干的章节。
- “评价 + 优化”：评价指标必须能服务决策变量或目标函数，否则只是装饰。
- “图像 + 物理”：像素特征必须转换为物理量或几何量，才能进入守恒或优化模型。
- “仿真 + 优化”：仿真负责评估不确定场景，优化负责给出可执行决策，二者要共用同一套输入参数。

## 分诊后立刻检查的下一步

- 建模：写出主线的变量、目标、核心约束、单位和第一版基准模型。
- 代码：按主线所需字段做 schema、缺失、异常、单位和样本覆盖检查。
- 写作：记录题面交付物、预期结果表、图表证据链和摘要中可能出现的核心数值。
- 复查：按评分维度检查是否存在漏交付、无完整表、无验证或无现实原型的问题。

这些是下一步检查项，不是默认多代理派发。只有用户明确要求拆分、派发或并行时，才把它们改写成 handoff。
