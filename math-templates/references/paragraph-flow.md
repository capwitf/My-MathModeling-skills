# 竞赛段落流

## 用途

当数学建模竞赛论文段落显得松散、重复或难以扫读时，使用本参考。适用章节包括问题分析、模型准备、模型建立、结果分析、灵敏度分析和模型评价。

## 核心原则

一个竞赛段落写得顺，意味着评委能迅速识别：

- 该段的唯一任务；
- 主题句；
- 支撑该段的公式、表格、图像或推理；
- 结论句或边界句；
- 它与当前子问题的关系。

## 读者检查

检查任意段落时，问：

1. 这一段在做什么：定义、建模、求解、报告、解释、检验还是限定？
2. 首句是否说出了这个任务？
3. 变量、单位、场景和基准是否不依赖隐藏上下文也能读懂？
4. 句与句之间是否通过原因、条件、对比、结果或解释连接？
5. 末句是可用结论，还是只起装饰性过渡？

## 分章节段落任务

| 章节 | 段落任务 |
| --- | --- |
| 问题分析 | `总述 -> **问题一：** 目标 -> 数学归类 -> 难点 -> 路线 -> 输出/验证` |
| 模型准备 | `共用对象 -> 变量/坐标 -> 基础关系 -> 后续用途` |
| 模型建立 | `现实约束 -> 变量/公式 -> 含义/单位 -> 适用范围` |
| 求解方法 | `问题类型 -> 算法选择 -> 参数/终止准则 -> 输出文件` |
| 结果分析 | `结论 -> 证据 -> 比较/机理 -> 回答题目` |
| 灵敏度分析 | `扰动设计 -> 变化范围 -> 指标响应 -> 稳定性结论` |
| 模型评价 | `难点转译/判定闭环/跨问递推 -> 证据或图表 -> 不足影响 -> 推广边界` |

## 修复动作

- 拆分混合了定义、公式推导和结果解释的段落。
- 将变量定义移到使用该变量的公式之前。
- 将"由表/图可见"放在图表已经引入之后，而不是放在读者还不知道图表内容之前。
- 用明确关系替换空泛过渡："因此"、"相比之下"、"在该约束下"、"进一步"、"这说明"。
- 当结果依赖特定场景、松弛条件、时间尺度或关键假设时，补一个边界句。
- 删除只表示下一节即将出现的句子。

## 好模式

```text
**问题二：** 问题二要求在问题一确定的运输网络基础上进一步降低总成本，因此可归结为带容量约束的路径优化问题。该问题的难点在于车辆容量、时间窗和节点需求同时限制可行路径。考虑到决策变量具有离散性，本文采用整数规划模型描述车辆选择和路径分配。模型输出为各车辆的服务路径和总成本，并通过容量约束回代与成本敏感性分析检验结果。
```

这段顺的原因：

- 首句说明目标和数学类型；
- 第二句说明难点；
- 第三句解释路线选择；
- 第四句说明输出和验证。

## 坏模式

```text
问题二: 比较复杂，需要建立优化模型。本文首先分析数据，然后建立模型，最后求解并进行灵敏度分析。该模型具有较好的效果。
```

问题：

- 使用 ASCII 冒号，而不是 `**问题二：** `；
- 没有数学类型；
- 没有具体难点；
- 没有输出；
- 没有证据或边界；
- "较好的效果"没有可衡量含义。
