# 竞赛论文语言护栏

## 用途

当建模逻辑已经明确、需要修正文风时使用本文件。它修语言，不替代证据；正文结论的强度不得超过模型、表格、图像或运行记录能够支持的范围。

## 基本语气

- 优先使用评委可读的精确表达，而不是文学化修饰。
- 先说明模型承担的任务，再命名具体方法。
- 数字、单位、场景和基准应靠近对应结论。
- 表述要谨慎但明确：说清结果说明什么，以及在哪些条件下成立。
- 不用漂亮句子掩盖推理薄弱。

## 推荐语体

使用技术说明书式中文：

- "为刻画..."
- "定义..."
- "建立..."
- "由式..."
- "代入..."
- "求得..."
- "结果表明..."
- "进一步检验..."
- "在...条件下..."

推荐：

```text
由表3可见，方案二的总成本为 12.6 万元，较基准方案下降 8.4%，说明约束重排后运输距离得到压缩。
```

避免：

```text
由表3可知，方案二效果较好，模型具有较强优越性。
```

## 分问标签格式

在 `问题重述`、`问题分析` 和简短分问总结中，使用加粗分问标签：

```markdown
**问题一：** [正文。]
**问题二：** [正文。]
```

规则：

- `问题一` 等分问标签加粗。
- 使用中文全角冒号 `：`。
- 加粗标签后保留一个可见空格。
- 不要使用 ASCII 冒号，如 `问题一:`。
- 不要写成 `问题一 ：`，冒号前不能有空格。

## 数字与单位

- 每个正文数字在必要时都要带单位、场景或分母。
- 百分比提升必须说明比较基准和分母。
- 正文、公式、表头和图轴中的单位应一致。
- 摘要、正文、表格和图中的同一数字不要随意采用不同四舍五入；若必须不同，应说明规则。
- 不要裸写"最优"等排序结论，除非可行性条件和比较范围已经明确。

## 结论强度

强结论需要强证据：

| 谨慎使用 | 更适合竞赛论文的写法 |
| --- | --- |
| 证明 | 表明 / 说明 / 验证了 |
| 显著优于 | 在[指标]上较[基准]提高[数值] |
| 完全可行 | 在[约束范围]内满足可行性条件 |
| 鲁棒性强 | 在[扰动范围]内[结论/排名]保持不变 |
| 具有广阔前景 | 可迁移至[同构场景]，需补充[参数/数据/约束] |

## 删除空话

- "本文将建立数学模型进行求解"
- "经过分析可知"
- "结果较为理想"
- "具有一定参考价值"
- "模型具有较强优越性"
- "由于时间有限"
- 没有说明合理性的"为了简化计算"
- 没有具体同构场景的"可推广到更多领域"

## 公式与图表语言

公式前：

```text
为刻画[现实关系]，定义[变量]表示[含义]，建立如下约束：
```

公式后：

```text
式(4)表示[现实含义]，其中[变量]的单位为[单位]，参数[参数]由[来源]确定。
```

图表前：

```text
为比较[对象]在[场景]下的[指标]差异，绘制图[编号]。
```

图表后：

```text
图[编号]显示，当[变量]变化时，[指标]呈现[趋势]，这说明[结论]。
```

## 人工智能与作者边界

若比赛要求披露 AI 使用情况，将披露内容放在官方表格、附录或指定支撑材料中。除非官方规则明确要求，不要把 AI 使用说明插入摘要或主体建模叙述。
